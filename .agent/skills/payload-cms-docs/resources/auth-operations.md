# Authentication Operations

Enabling Authentication on a Collection automatically exposes specialized operations across the Local API, REST API, and GraphQL API.

## Core Operations

### Access
Returns the permissions for the currently logged-in user. This identifies what collections and globals they can create, read, update, or delete.

- **REST**: `GET /api/access`
- **GraphQL**: `query { Access { ... } }`

### Me
Returns the current user's document and their JWT expiration time. Returns `null` if not authenticated.

- **REST**: `GET /api/[collection-slug]/me`
- **GraphQL**: `query { meUser { ... } }`

### Login
Authenticates a user via email/username and password. On success, returns the user document and a JWT. In REST/GraphQL, it also sets an **HTTP-only cookie**.

- **REST**: `POST /api/[collection-slug]/login`
- **Local**: `payload.login({ collection: 'users', data: { email, password } })`
- **GraphQL**: `mutation { loginUser(email, password) { ... } }`

### Logout
Invalidates the current session. Since cookies are HTTP-only, this operation must be called to safely clear the auth state.

- **REST**: `POST /api/[collection-slug]/logout`
- **GraphQL**: `mutation { logoutUser }`
- **Options**: `allSessions: true` will log the user out of all devices.

### Refresh
Generates a new JWT and clears/resets the HTTP-only cookie. Requires a non-expired token to function.

- **REST**: `POST /api/[collection-slug]/refresh-token`
- **GraphQL**: `mutation { refreshTokenUser { ... } }`

## Account Management

### Verify by Email
Activates a user account using a verification token sent via email.

- **REST**: `POST /api/[collection-slug]/verify/[token]`
- **Local**: `payload.verifyEmail({ collection: 'users', token })`

### Forgot Password
Generates a reset-password token and sends an email to the user.

- **REST**: `POST /api/[collection-slug]/forgot-password`
- **Local**: `payload.forgotPassword({ collection: 'users', data: { email } })`

### Reset Password
Updates the user's password using the token generated by the "Forgot Password" operation.

- **REST**: `POST /api/[collection-slug]/reset-password`
- **Data**: `{ token: '...', password: '...' }`

### Unlock
Manually unlocks a user who has been locked out due to too many failed login attempts.

- **REST**: `POST /api/[collection-slug]/unlock`
- **Local**: `payload.unlock({ collection: 'users' })`
